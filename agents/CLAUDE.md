# 공통 에이전트 가이드라인

## 공통 규칙

- 한국어로 응답한다.
- 추측하지 않는다. 코드에서 확인된 사실만 보고한다.
- 파일 경로와 라인 번호를 항상 포함한다.
- 응답은 구조화된 형식(마크다운)으로 작성한다.
- 민감 정보(API 키, 비밀번호, 토큰)는 절대 출력하지 않는다.

## 성능 규칙

- 불필요한 도구 호출을 최소화한다. 한 번의 검색으로 충분하면 추가 검색하지 않는다.
- 에이전트 타임아웃이 존재함을 인지하고, 충분한 정보를 수집했으면 즉시 결과를 반환한다.
- 파일 전체를 읽지 않는다. 관련 부분만 부분 읽기(offset, limit)로 확인한다.

## 출력 형식

모든 서브에이전트는 다음 JSON 형식으로 결과를 반환한다:

```json
{
  "status": "success" | "partial" | "failed",
  "summary": "한 줄 요약",
  "details": "상세 내용 (마크다운)",
  "references": [
    { "file": "파일경로", "line": 라인번호, "snippet": "관련 코드 조각" }
  ],
  "confidence": "high" | "medium" | "low"
}
```

## 실패 시 출력 규칙

에러가 발생하거나 작업을 완료할 수 없는 경우에도 반드시 위 JSON 형식으로 결과를 반환한다. 절대 빈 출력이나 비정형 텍스트를 반환하지 않는다.

```json
{
  "status": "failed",
  "summary": "실패 원인 한 줄 요약",
  "details": "에러 상세 내용",
  "references": [],
  "confidence": "low"
}
```
